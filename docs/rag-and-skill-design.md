# RAG and Skill Design

## RAG 设计

JobFit 在题目生成环节加入 RAG 流程，目标是让面试题更真实、更贴近岗位，而不是完全依赖大模型凭空生成。

流程：

```text
岗位和能力输入
  -> 生成搜索词
  -> 公开题源检索
  -> 网页正文抓取
  -> AI 抽取真实面试题
  -> 题目清洗与去重
  -> 能力标签匹配
  -> 题库沉淀
  -> 模拟面试召回
```

## 题库结构

题库按题目级记录存储，而不是按岗位文件夹硬拆。

每道题保留：

- 题干
- 岗位族
- 适用岗位
- 能力标签
- 题型
- 难度
- 来源 URL
- 原文片段
- AI 生成的考察点、答题框架和参考回答示例

这种结构支持跨岗位复用。例如跨部门沟通、风险处理、数据分析等能力可以被多个岗位共享。

## Skill 设计

JobFit 将高频 AI 判断能力沉淀为 Skill。

### 岗位能力模型 Skill

负责：

- 岗位族判断
- 职责抽取
- 核心能力建模
- 关键技能归一
- 面试关注点生成
- 不确定信息处理

### 题目筛选与改写 Skill

负责：

- 从公开资料中抽取真实面试题
- 判断题目适用岗位和能力标签
- 判断题型和难度
- 轻度改写题干
- 生成考察点、答题框架和参考回答示例
- 保留来源信息

## 产品价值

RAG 解决“题目从哪里来”的问题，Skill 解决“AI 如何稳定判断”的问题。

两者结合后，JobFit 不只是生成内容，而是能基于真实资料、业务规则和用户目标完成可追溯、可复用、可持续优化的面试训练流程。

# JobFit 题目筛选与改写 Skill

## 目标

将公开资料、网页正文或题库候选题转化为高质量 JobFit 面试练习题。这个 Skill 要保证题目来源真实、岗位匹配准确、能力标签可用，并能为每道题生成考察点、答题框架和参考回答示例。

核心原则：

- 真实公开资料负责提供题目来源。
- AI 负责抽题、筛选、打标签、轻度改写和生成答题参考。
- 题目和答案必须区分来源：题目来自真实资料，答题建议可由 AI 生成。

## 输入

- `targetJobTitle`：当前目标岗位。
- `jobFamily`：岗位族。
- `coreAbilities`：岗位核心能力。
- `gapAnalysis`：可选，用户能力差距。
- `sourceUrl`：可选，公开资料 URL。
- `sourceText`：可选，网页正文或资料片段。
- `existingQuestions`：可选，本地题库候选题。
- `userProfile`：可选，用户背景，用于生成更贴近用户经历的参考回答。

## 输出

返回题目级记录：

```json
{
  "id": "稳定题目 ID",
  "title": "标准化题干",
  "jobFamily": "岗位族",
  "jobTitles": ["适用岗位"],
  "abilityTags": ["能力标签"],
  "questionType": "题型",
  "difficulty": "easy | medium | hard",
  "priority": "high | medium | low",
  "sourceUrl": "来源 URL",
  "sourceExcerpt": "支持该题的原文片段",
  "sourceNote": "来源说明",
  "examPoint": "这道题考察什么",
  "answerFramework": "建议答题结构",
  "sampleAnswer": "AI 辅助生成的参考回答示例",
  "isSourceBacked": true
}
```

字段规则：

- `abilityTags`：1-3 个，优先来自当前岗位能力模型。
- `questionType`：必须使用题型分类表。
- `sourceExcerpt`：不能过长，只保留能证明题目来源的片段。
- `sampleAnswer`：如果来源没有答案，可由 AI 生成，但必须作为辅助示例。

## 工作流

1. 从来源正文中识别真实面试题。
2. 过滤广告、评论、导航、泛建议和非题目内容。
3. 在不改变原始意图的前提下标准化题干。
4. 判断题目所属岗位族和适用岗位。
5. 根据岗位能力模型和题目意图打能力标签。
6. 判断题型。
7. 评估难度和训练优先级。
8. 必要时生成考察点、答题框架和参考回答。
9. 保留来源 URL 和原文片段，保证题目可追溯。
10. 拒绝低置信度题目，不为了凑数强行输出。

## 题型分类

优先使用以下题型：

- `experience`：经验题，要求讲过往工作、项目或案例。
- `case`：案例题，要求处理一个具体业务场景。
- `behavioral`：行为题，考察协作、冲突、沟通、压力或决策行为。
- `technical`：专业题，考察工具、方法、专业知识或执行能力。
- `analysis`：分析题，考察诊断、推理、数据解读、取舍或策略判断。
- `stress`：压力/异常题，考察失误、风险、突发问题、挫折或压力处理。
- `motivation`：动机题，考察求职动机、职业选择、岗位匹配和自我认知。

## 标签判断规则

- 能力标签优先来自当前岗位能力模型。
- 判断题目归属时，不只看关键词，要看题目意图和业务场景。
- 一个题可以适配多个岗位，但必须有一个主岗位族。
- 不要因为题目出现“财务审批”“成本”就把采购题归为财务题。
- 不要给普通财务题强加 SQL、BI、Python 标签，除非题目明确考察数据工具。
- 跨岗位复用只允许发生在真实共享能力上，例如跨部门沟通、风险处理、项目推进。

## 改写规则

- 保留原题意图，不改变考察方向。
- 可以让题干更清晰、更适合练习。
- 不要把来源中没有的答案伪装成原网页答案。
- 如果来源只有题目，没有答案，AI 可以生成考察点、答题框架和参考回答示例。
- 页面或数据结构必须能区分“真实题源”和“AI 辅助生成答案”。

## 答题参考生成规则

每道题的答题参考分为三层：

- `examPoint`：考察点，说明面试官想看什么能力。
- `answerFramework`：答题框架，例如 STAR、背景-动作-结果、问题-分析-方案-结果、方法-案例-反思。
- `sampleAnswer`：参考回答示例，根据岗位和用户背景生成更完整的一版。

## 拒绝入库规则

出现以下情况时，不允许入库：

- 不是面试题。
- 无法绑定岗位或能力。
- 过于泛泛，不能训练具体能力。
- 来源正文噪音过大，无法支持题目真实性。
- 与已有题高度重复，且没有新的岗位、能力或场景价值。

## 异常处理

- 如果真实题源抽取失败，返回空列表并说明原因，不要凭空补题。
- 如果本地题库匹配弱，不要强行塞入当前岗位。
- 如果缺少用户背景，参考回答可以基于岗位生成，但不要假造用户经历。

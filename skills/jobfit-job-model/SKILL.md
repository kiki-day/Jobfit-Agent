# JobFit 岗位能力模型 Skill

## 目标

将目标岗位信息转化为结构化岗位能力模型，为后续个人能力画像、能力差距分析、题目筛选、模拟面试和回答点评提供统一能力基准。

这个 Skill 的核心不是“总结 JD”，而是把招聘要求拆成可评估、可出题、可评分的能力结构。

## 输入

- `targetJobTitle`：目标岗位名称。
- `jobDescription`：岗位 JD、职责、任职要求或招聘文本。
- `industry`：行业或业务领域，可为空。
- `jobDirection`：岗位方向，可为空。
- `levelRequirement`：经验要求或职级，可为空。
- `extraInfo`：用户补充信息，可为空。

## 输出

必须输出以下结构：

```json
{
  "isEnough": true,
  "title": "标准化岗位名称",
  "jobFamily": "岗位族",
  "industry": "行业",
  "level": "经验/职级",
  "responsibilities": ["核心职责"],
  "coreAbilities": ["核心能力"],
  "keySkills": ["关键技能"],
  "interviewFocus": ["面试关注点"],
  "abilityWeights": [
    {
      "ability": "能力名称",
      "weight": 20,
      "reason": "为什么重要"
    }
  ],
  "uncertainties": ["不确定项"]
}
```

字段要求：

- `responsibilities`：4-8 条，必须是具体工作职责。
- `coreAbilities`：4-7 条，必须能用于后续能力差距分析。
- `keySkills`：6-12 条，包含工具、方法、流程、专业知识或业务要求。
- `interviewFocus`：4-7 条，必须能转化为面试考察点。
- `abilityWeights`：能力权重总和建议接近 100。
- `uncertainties`：信息不足或岗位冲突时必须标记。

## 工作流

1. 判断岗位信息是否足够分析。
2. 优先根据 JD 内容判断岗位族，再参考岗位名称。
3. 从动作、产出、系统、流程和协作对象中抽取核心职责。
4. 将职责转化为可评估的核心能力。
5. 只补充与当前岗位相关的技能、工具和方法。
6. 基于职责和能力生成面试关注点。
7. 根据岗位重要性和面试高频程度分配能力权重。
8. 对信息不足或冲突的部分标记不确定项，不凭空补全。

## 岗位族判断规则

- 如果岗位名称和 JD 冲突，优先相信 JD 内容。
- 财务类岗位重点识别会计处理、费用合规、结账、报表、发票、税务协同和准确性。除非 JD 明确出现财务分析、BI、SQL、数据分析、经营分析、商业分析，否则不要加入 SQL、BI、Python、数仓等能力。
- 采购类岗位重点识别供应商管理、寻源比价、成本控制、合同执行、交付协调和风险控制。
- 设计类岗位需要区分视觉设计、交互设计、产品设计、品牌设计、电商设计，不要把纯视觉岗位强行套入产品经理能力。
- 产品类岗位重点识别需求分析、用户洞察、方案设计、优先级判断、数据化决策和跨部门推进。
- 研发类岗位重点识别技术实现、系统设计、问题排查、质量保障、协作交付。
- 销售类岗位重点识别客户开发、线索判断、沟通推进、商务谈判、转化跟进和 CRM 纪律。
- 运营类岗位重点识别流程执行、活动策划、数据观察、跨部门协同、增长或留存动作。

## 能力建模规则

- 核心能力必须来自 JD、岗位族常识或明确业务场景。
- 不输出泛泛的“沟通能力”“学习能力”，除非说明对应岗位场景。
- 每项能力都应该可以用于后续出题和评分。
- 关键技能必须具体，可作为题目标签或检索标签。
- 职责描述工作内容，不描述性格特质。
- 证据弱时进入 `uncertainties`，不要强行补全。

## 异常处理

- 如果输入是问候、闲聊或与岗位无关，返回 `isEnough: false`，并要求补充目标岗位或 JD。
- 如果只有岗位名称，可以生成保守模型，但必须在 `uncertainties` 标记缺少 JD。
- 如果岗位方向不清晰，给出最可能岗位族，同时列出不确定点。

## 示例

输入：

```json
{
  "targetJobTitle": "财务专员",
  "jobDescription": "负责凭证录入、费用报销审核、月度结账、税务申报支持。"
}
```

预期能力方向：

- 会计准则理解与账务处理
- 费用合规审核
- 月度结账执行
- 财务报表整理
- 税务申报协同
